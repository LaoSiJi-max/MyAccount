# MyAccount

[中文](README.md) | [日本語](README_ja.md)

ASP.NET MVCとEntity Frameworkを使用して開発した、顧客残高・入出金履歴管理システムです。

## プロジェクト紹介

大学時代にASP.NET MVCを学びながら、少しずつ作っていった会計管理システムです。

当時は毎日のようにルームメイトと一緒に食事をしており、私がまとめて支払い、それぞれが使った金額を記録していました。ルームメイトからは一定期間ごとにまとめてお金を受け取っていたため、MVCアーキテクチャを学ぶついでに、この帳簿を管理する小さなシステムを作ってみようと思いました。

学期の初めから少しずつ機能を追加し、期末まで開発を続け、最終的には授業の期末課題として先生に提出しました。

実際の帳簿管理については、その後すっかり使わなくなりました。数人分の記録であれば、Excelのほうがずっと便利だったからです。（￣▽￣）

このプロジェクトは期末課題として完成させただけでなく、MVCアーキテクチャについて多くのことを学ぶ機会にもなりました。そして、すべてを最初から自分で作り直す必要はない、ということも少しずつ理解するようになりました。

既存の成果や道具を理解し、その上でさらに先へ進むことのほうが、より価値のある場合も多いと思います。

## 主な機能

### 顧客管理

* 顧客アカウント一覧の表示
* 条件を指定した顧客検索
* 顧客情報の追加・変更・削除
* 顧客一覧のページング
* 顧客ごとの現在残高の確認
* 検索結果のExcel出力

### 入出金管理

* 指定した顧客への入出金記録の追加

* 金額と備考の入力
* 顧客ごとの過去の取引履歴の確認
* 全取引ログの検索・確認
* 取引記録の取消し
* 取り消した取引の復元
* 取消し・復元時の顧客残高の自動修正

## システム構成

ASP.NET MVCの基本構成を採用し、Controllerとデータアクセス処理の間にService層を設けています。

```text
ブラウザ
  │
  ▼
Razor View
  │
  ▼
Controller
  │
  ▼
Service ビジネスロジック
  │
  ▼
Entity Framework
  │
  ▼
SQL Server LocalDB
```

各部分の主な役割は以下のとおりです。

* **View**
  Razorテンプレートを使用し、顧客、取引履歴、管理者、入出金記録などの画面を表示します。

* **Controller**
  ブラウザからのリクエストを受け取り、画面遷移を制御し、必要なビジネスロジックを呼び出します。

* **Service**
  顧客管理、入出金処理、残高変更、取引取消し、管理者認証などの業務処理を担当します。

* **Model**
  顧客、管理者、取引種別、取引記録などのデータ構造を定義します。

* **Entity Framework**
  オブジェクトとデータベースのマッピング、およびデータの永続化を担当します。

* **SQL Server LocalDB**
  顧客情報、管理者情報、取引種別、取引履歴を保存します。

## アーキテクチャ上の試み

### MVCの役割分担

画面表示、リクエスト処理、データモデルをそれぞれView、Controller、Modelに分け、すべてのコードを画面側に集中させない構成を目指しました。

### Service層

主要な業務機能についてServiceインターフェースと実装クラスを定義し、入出金処理や顧客管理のロジックをControllerから分離しています。

当時はまだ依存性注入コンテナを本格的に導入していませんでしたが、Controllerにビジネスロジックを集中させないようにする考え方を試していました。

### 独自の権限チェック

独自の `AuthorizeAttribute` を使用して、管理者のログイン状態と権限レベルを確認しています。

未ログインの利用者や、必要な権限を持たない管理者が特定のページへアクセスすることを制限します。

### 残高と取引履歴の整合性

入出金記録を追加すると、取引履歴を保存すると同時に顧客残高も更新します。

取引を取り消す場合は、取引状態を変更するだけでなく、元の金額が残高へ与えた影響を逆算して顧客残高を修正します。取り消した取引を復元した場合は、その影響を再び残高へ反映します。

この部分では、単純なデータの追加・変更・削除だけでなく、取引状態と残高の整合性も考慮しました。

### Code Firstによるデータベース構築

Entity Framework Code Firstを使用し、C#で定義したデータモデルをもとにデータベース構造を生成しています。

初回のデータベース初期化時には、既定の管理者データも作成します。

### ページングとExcel出力

顧客一覧と取引履歴一覧はページングに対応しています。

また、顧客の検索結果をExcelファイルとして出力できます。

## 主なデータモデル

### Customer

顧客情報には主に以下の項目があります。

* ID
* 顧客番号
* 顧客名
* ログインパスワード
* 現在残高
* 最終操作日時

### Owner

管理者情報には主に以下の項目があります。

* ID
* ログイン名
* パスワード
* 管理者権限レベル
* アカウント状態

### Account Type

取引種別には主に以下の項目があります。

* ID
* 取引種別名
* 残高への影響方法
* 取引種別の説明

例：

* 収入
* 支出
* 残高調整

### Account Log

取引記録には主に以下の項目があります。

* ID
* 顧客情報
* 取引種別
* 金額
* 備考
* 操作日時
* 取消し状態

## 入出金処理の流れ

```text
顧客を選択
   │
   ▼
取引種別を選択
   │
   ▼
金額と備考を入力
   │
   ▼
取引記録を作成
   │
   ▼
顧客残高を変更
   │
   ▼
データベースへ保存
```

取引を取り消す場合：

```text
取引記録を選択
   │
   ▼
現在の取引状態を確認
   │
   ▼
元の取引による影響を逆算
   │
   ▼
顧客残高を修正
   │
   ▼
取引を取消し状態に変更
```
